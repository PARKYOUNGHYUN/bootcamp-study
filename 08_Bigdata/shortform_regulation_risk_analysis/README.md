## 3. 사용 변수

> 설문 데이터에는 다양한 변수가 존재하지만
> 숏폼 사용 패턴과 직접적으로 관련된 변수만 선별하여 모델을 구성했습니다.

### Feature (입력 변수 — 원인)

| 변수명 | 설문 문항 | 척도 |
|---|---|---|
| Q7_1A | 숏폼이 온라인 동영상 서비스 이용에서 차지하는 비중 | 1~4점 |
| Q7_3A | AI 알고리즘에 의해 같은 유형의 숏폼 콘텐츠를 반복해서 보게 된다 | 1~4점 |
| Q3_3 | SNS 이용시간 조절에 대한 어려움 | 1~4점 |

> Q14_1 (장시간 이용 경험) 변수는 초기 분석에 포함했으나
> 상관계수 0.28, Feature Importance 3.0%로 기여도가 낮았습니다.
> Q14_1 제외 후 Random Forest 기준 Recall이 0.69 → 0.71로 오히려 향상되어 최종 제외했습니다.

> **Q3_3 포함 이유**
> 두 변수는 측정 대상이 다릅니다.
> 'SNS 조절 어려움(Q3_3)'은 인스타그램, 트위터 등 SNS 전반의 이용시간 조절 어려움을 의미하며,
> '숏폼 조절 어려움(Q7_2A)'은 숏폼 콘텐츠 시청에 특화된 조절 어려움을 의미합니다.
> 따라서 본 연구에서는 일반적인 SNS 사용 습관이 숏폼 시청 조절 어려움과 어떤 관계를 가지는지
> 확인하기 위해 해당 변수를 Feature로 포함했습니다.

### Target (출력 변수 — 결과)

| 변수명 | 설문 문항 | 변환 |
|---|---|---|
| Q7_2A | 본인의 의지대로 숏폼 시청을 조절하는 것에 느끼는 어려움 | 1~2점 → 0 (안전) / 3~4점 → 1 (위험) |

## 4. 전처리

| 항목 | 처리 방법 | 이유 |
|---|---|---|
| 연령 필터링 | 10~39세만 추출 | 팀 전체 분석 대상 연령대 통일 |
| Q7_1A, Q7_3A, Q7_2A 결측값 | 제거 (dropna) | 숏폼 비이용자 → 분석 대상 아님 |
| Q3_3 결측값 | 제거 (dropna) | 응답 없는 행 제거 |
| Q7_2A Target 변환 | 1~2점 → 0 / 3~4점 → 1 | 이진 분류 모델 적용 위해 |
| 학습/테스트 분리 | train_test_split (8:2, random_state=42) | 모델 학습 및 성능 평가를 위해 분리 |

### 전처리 후 데이터 비율

```
안전 (0) = 약 62%
위험 (1) = 약 38%
→ 극단적인 클래스 불균형(Class Imbalance)이 존재하지 않아
  별도의 오버샘플링(SMOTE 등)이나 가중치 조절 없이 진행함
```

## 6. ML 모델 결과

### 모델 비교

| 항목 | Logistic Regression | Random Forest | XGBoost | RF Tuned (HyperOpt) | Stacking |
|---|---|---|---|---|---|
| 안전(0) F1 | 0.83 | **0.84** | **0.84** | 0.83 | **0.84** |
| 위험(1) F1 | 0.72 | **0.74** | **0.74** | **0.74** | **0.74** |
| 위험(1) Recall | 0.67 | 0.71 | 0.70 | **0.72** | 0.70 |
| 위험(1) Precision | 0.78 | 0.77 | **0.78** | 0.76 | **0.78** |

> 본 연구의 목적은 위험군 탐지이므로
> False Negative(위험군을 놓치는 경우)를 줄이는 것이 중요합니다.
> 따라서 Recall을 주요 지표로 평가했습니다.

---

### 모델별 시도 결과

**Logistic Regression (베이스라인)**
```
가장 단순한 선형 모델로 베이스라인 설정
위험군 Recall 0.67 → 위험군 탐지 성능 부족
```

**Random Forest**
```
위험군 Recall 0.71로 베이스라인 대비 향상
기본 파라미터에서 안정적인 성능 확인
```

**XGBoost**
```
Random Forest와 유사한 성능 (Recall 0.70, F1 0.74)
부스팅 계열 모델도 동일 수준임을 확인
```

**HyperOpt (Random Forest 하이퍼파라미터 튜닝)**
```
베이지안 최적화로 50회 탐색 수행
최적 파라미터: n_estimators=100, max_depth=7,
              min_samples_split=10, min_samples_leaf=4
→ 위험군 Recall 0.72로 기본 RF(0.71) 대비 소폭 향상
→ 최종 모델로 선택
```
> 베이지안 최적화를 통해 50회 튜닝을 진행했으나 F1-score 기준 추가적인 향상은 없었습니다.
> 다만 Recall이 0.71 → 0.72로 소폭 향상되어 위험군 탐지 목적에 더 적합한 것으로 판단했습니다.

**Stacking (LR + RF + XGBoost → LR 메타 모델)**
```
Precision 0.78로 향상되었으나
Recall이 0.70으로 RF 대비 낮음
→ 성능 향상이 없어 복잡도 대비 이점 없음
→ 본 연구 목적(위험군 탐지)에서 RF 대비 우위 없음
```

---

### 최종 모델 선택: RF Tuned (HyperOpt)

```
5가지 방법을 비교한 결과
RF Tuned가 위험군 Recall 0.72, F1 0.74로 가장 안정적
```

> 본 연구의 목적은 위험군 탐지이므로 Recall을 최우선 지표로 고려했습니다.
> RF Tuned가 Random Forest 대비 Recall 0.71 → 0.72로 소폭 향상되었으며
> F1-score는 동일(0.74)하여 RF Tuned를 최종 모델로 선택했습니다.
> HyperOpt 베이지안 최적화를 통해 도출된 파라미터를 적용한 결과입니다.

### Feature Importance

| 변수 | 중요도 |
|---|---|
| Q7_3A (AI 알고리즘 반복 시청) | **47.2%** |
| Q7_1A (숏폼 비중) | 34.4% |

```
→ EDA에서 확인한 상관관계 순위와 Feature Importance 순위가 유사하게 나타났다
→ AI 추천 알고리즘 관련 인식이 가장 높은 예측 중요도를 보임
```

#### 모델 해석 주의사항
> ※ RF Tuned의 Feature Importance는 변수의 예측 기여도를 의미하며
> 인과관계를 직접적으로 의미하지는 않는다.
> Feature Importance는 변수의 분할 기준으로 사용된 빈도와 정보 이득 감소량을 기반으로 계산됩니다.

#### 해석 결론
> 위험군을 예측하는 데 있어 AI 알고리즘 관련 인식 변수가 가장 강한 예측 신호로 나타났다.

#### 배경 설명
> AI 추천 알고리즘은 사용자가 관심을 보인 콘텐츠와 유사한 영상을 지속적으로 노출한다.
> 이로 인해 사용자가 의도하지 않아도 연속 시청이 발생할 가능성이 있다.
> 이는 숏폼 조절 어려움과의 높은 상관관계를 부분적으로 설명할 수 있다.

### 입력값

```
숏폼 비중 선택        (0~25% / 25~50% / 50~75% / 75% 이상)
AI 반복 시청 정도     (전혀 그렇지 않다 ~ 매우 그렇다)
SNS 조절 어려움       (전혀 그렇지 않다 ~ 매우 그렇다)
```

### 출력값

```
위험 점수 계산 방식
확률 × 100 = 위험 점수
예) predict_proba = 0.73 → 위험 점수 73점

실제 기준값 (예측 확률 분포 3등분 기반)
🔴 60점 이상  → 위험 — 숏폼 사용 패턴을 조절할 필요가 있습니다
🟡 12~59점   → 주의 — 사용 습관을 점검해보세요
🟢 12점 미만  → 안전 — 건강한 숏폼 사용 패턴입니다
```

> 위험 기준 근거: 예측 확률 분포를 3등분하여
> 하위 33%를 안전(기준 12점), 중간 33%를 주의, 상위 33%를 위험(기준 60점)으로 설정했습니다.
